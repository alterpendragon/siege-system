# CS50x Final Project Rules & Academic Honesty - Siege System

## 1. Core Project Requirements
* **Scope & Nature:** The project must entail more time and effort than a standard problem set. It can be written in any language and must draw upon the course's lessons to solve an actual problem.
* **Originality:** The essence of the work must be entirely my own. The overarching philosophy of the course is "be reasonable."

## 2. Submission Deliverables
To successfully receive the CS50x Certificate, the following strict deliverables must be met:
* **Video Demonstration:** A maximum 3-minute video uploaded to YouTube. It MUST begin by displaying: Project Title, My Name, GitHub/edX usernames, City/Country, and the date of recording. The remainder must demonstrate the software in action.
* **README.md:** A comprehensive documentation file (target: ~750 words). It must contain:
    * The Project Title and Video Demo URL at the top.
    * A detailed description of the project.
    * An explanation of what every submitted file contains and does.
    * A breakdown of the design choices and architectural decisions made during development.
* **Code Submission:** Executed via the `submit50` command within the project directory.

## 3. Academic Honesty & The Socratic Boundary
CS50x includes a specific exception for the final project regarding AI tools, but I am setting strict personal boundaries to ensure genuine learning:

* **The AI Exception:** For the final project only, CS50x allows the use of external AI tools (ChatGPT, Claude, Gemini). However, these tools must *amplify*, not *supplant*, my productivity.
* **The Socratic Method:** AI will act strictly as a Tech Lead and tutor. It is permitted to:
    * Review architectural decisions and database schemas.
    * Explain abstract concepts (e.g., "How do API pagination limits work?").
    * Assist in debugging by pointing out the *location* or *nature* of an error.
* **No Blind Copy-Pasting:** The AI will not be used to write the core application logic outright. I must be able to explicitly explain how and why every single line of code in the Siege System works.
* **Mandatory Citation:** Any use of AI tools for code structure, or any external code snippets (e.g., from documentation or forums), MUST be explicitly cited in the comments of the source code, as required by the Academic Honesty policy.
